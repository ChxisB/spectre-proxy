## VERIFICATION GUARANTEE (NON-NEGOTIABLE)

**NOTHING is "done" without PROOF it works.**

### Pre-Implementation: Scenario Contract

BEFORE writing ANY code, define scenarios covering:
| Class | Required |
|---|---|
| Happy path | yes |
| Edge/boundary/empty/malformed | yes |
| Adjacent-surface regression | yes |

Each scenario MUST specify, upfront:
- Pass condition as a binary observable ("returns 200 + body matches schema"), not "should work"
- The REAL surface that proves it: curl, test output, CLI stdout, etc.
- The test file + test name that exercises this scenario (written test-first)

These scenarios are the CONTRACT. You are not done until every one PASSES with evidence.

### Execution & Evidence Requirements

Every scenario requires captured proof:
| Artifact | Source | Captures |
|---|---|---|
| RED→GREEN proof | Test runner output before AND after the change | Test id + assertion message in both states |
| Surface artifact | CLI / curl / browser | What the user actually sees |

Tests are the FLOOR (always required). Surface artifact is the CEILING (also required). "tests pass" alone is NOT done.

### MANUAL QA MANDATE

**YOU MUST EXECUTE MANUAL QA YOURSELF. THIS IS NOT OPTIONAL.**

| If your change... | YOU MUST... |
|---|---|
| Adds/modifies a CLI command | Run the command with bash. Show the output. |
| Changes build output | Run the build. Verify output files exist. |
| Modifies API behavior | Call the endpoint. Show the response. |
| Changes UI rendering | Verify visible output. |
| Adds a new tool/hook/feature | Test it end-to-end in a real scenario. |
| Modifies config handling | Load the config. Verify it parses correctly. |

**UNACCEPTABLE QA CLAIMS:**
- "This should work" - RUN IT.
- "The types check out" - Types don't catch logic bugs. RUN IT.
- "Tests pass" - Tests cover known cases. Does the ACTUAL FEATURE work? RUN IT.

### TDD Workflow (MANDATORY on every production change)

Test-first is not optional. Every behavior change follows RED -> GREEN -> SURFACE.
1. **RED**: Write the failing test FIRST. Run it. Capture the assertion message proving it fails.
2. **GREEN**: Write the SMALLEST change that flips RED->GREEN. Re-run. Capture GREEN output.
3. **SURFACE**: Exercise the real user-facing surface. Capture output.
4. **REGRESSION**: Re-run the FULL scenario list. Record PASS/FAIL.
